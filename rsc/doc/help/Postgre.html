This document explains the default database setup of the PostgreSQL Server used with the Facial Recognition System. Each table has a "Default Usage" section which is shows the column names and the datatypes for reference when manipulating the tables from the Manage GUI.

1.Access

This table contains the pairings of the Users and the Rooms. A user (referenced by UserID) is given access to a room (referenced by RoomID). The default example is (UserID:10000 and RoomID:100), this allows the User with the ID "10000" access to the Room with the ID "100"

Default Usage:

("UserID","RoomID")

(Integer,Integer)


2.Building

This table contains the buildings of the company. Each building has an ID, a Name and some information. A building can have multiple departments.

Default Usage:

("BuildingID","BuildingName","BuildingInfo")

(Integer,VarChar,VarChar)


3.Camera

This table contains a list of cameras used for the Facial Recognition System. Each camera has an ID, a Name, a room ID which relates to the room the camera is securing. The camera also has an IP Address which is how the system connects to cameras on the network and there is some Camera Information.

Default Usage:

("RoomID","CameraName","CameraIPAddress","CameraID","CameraInfo")

(Integer,VarChar,VarChar,Integer,VarChar)


4.Department

This table contains the company's departments. Each department has an ID, a Name, department information and each department has a building ID which relates to a building. A department is within a building. A department can have many rooms.

Default Usage:

("DepartmentID","BuildingID","DepartmentName","DepartmentInfo")

(Integer,Integer,VarChar,VarChar)


5.Room

This table contains the company's rooms. Each Room has an ID, a Name, room information and a department ID which relates to a department. A room will be located within a department.

Default Usage:

("RoomID","DepartmentID","RoomNumber","RoomInfo")

(Integer,Integer,VarChar,VarChar)


6.User Credentials

This table contains the user credentials for access to this Facial Recognition System's collection of programs. Each user has an ID, username (which is used for logging into the system), a password (which is hashed, this hashed password is verified against the user input password), user access level which determins which programs the user has access to (5=Master[Admin], 4=Manage, 3=Monitor, 2=Enrol, 1=Security, 0=No Access). A user from a higher level can access lower level programs aswell. Each user also has a user information section.

Default Usage:

("UserID","UserAccessLevel","Password","UserName","UserInfo")

(Integer,Integer,VarChar,VarChar)


7.User Face

This table contains users enrolled into the Facial Recognition System. Each user has a ID, first name and last name. Note that the actual parings between users and Room doors occur in the 'Access' table.

Default Usage:

("UserID","UserName","UserSurname")

(Integer,VarChar,VarChar)
